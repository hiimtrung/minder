# Phase 6 Tracker — Branch Topology, IDE Bootstrap, Secure Installer Automation, and Update Delivery

**Goal**: move CLI-side branch-topology discovery plus repository-local IDE bootstrap, secure MCP/instruction/agent installation, cross-platform installer automation, and explicit update-check/self-update flows into a dedicated post-Phase-5 track so Minder can provision repo-scoped LLM integration without diluting the current transition focus on cross-repo impact traversal.

## Status

| Area                            | Status        | Notes                                                                                        |
| ------------------------------- | ------------- | -------------------------------------------------------------------------------------------- |
| Phase 6                         | `IN PROGRESS` | Now narrowed to CLI completion and first-release preparation                                 |
| Repo-local IDE bootstrap        | `IN PROGRESS` | CLI now installs repo-local MCP config plus instruction and agent files                      |
| Secure installer distribution   | `IN PROGRESS` | Release workflow exists; cross-platform installer hardening remains                          |
| Update delivery and self-update | `BACKLOG`     | Explicit update check and self-update flow still needs to be shipped for both CLI and server |

## Scope Note

This phase carries forward the old transition backlog item `AT-T10` and extends the follow-up work into repository-local IDE bootstrap, installer automation, and release-update ergonomics.

The transition tracker has already closed runtime search, impact, and prompt-context follow-up in `AT-T11`, while this phase now owns only CLI/repo-scanner automation, repo-scoped IDE integrations, first-release installer/distribution work, and explicit update-check/self-update flows.

Non-CLI and deferred post-release backlog has been moved into [phase-7-learning-and-advanced-runtime.md](phase-7-learning-and-advanced-runtime.md).
